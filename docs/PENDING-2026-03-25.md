## CRÍTICO — ArgoCD em Unknown (todos os apps)

### Sintoma
Todos os ArgoCD Applications estão com `sync.status: Unknown`. O problema surgiu após
o pod `argocd-redis` reiniciar (~5h antes da sessão de 2026-03-25), rotacionando a senha
e deixando o `argocd-application-controller` com erro `NOAUTH Authentication required`.

Reiniciar os componentes limpou o erro de NOAUTH nos logs, mas o status Unknown persiste.

### Impacto
- ArgoCD não faz auto-sync. Mudanças no `rbx-infra` não se propagam automaticamente.
- **Workaround atual:** aplicar manualmente com `kubectl apply -k apps/prod/<produto>/`

### Diagnóstico
```bash
kubectl --kubeconfig ~/.kube/config-rbx -n argocd get applications
kubectl --kubeconfig ~/.kube/config-rbx -n argocd logs argocd-application-controller-0 --tail=50
```

### Possíveis causas a investigar
1. O `argocd-application-controller` pode estar usando endereço de repo-server obsoleto (ClusterIP antigo).
2. A versão do ArgoCD pode estar com bug conhecido de recovery pós-Redis restart.
3. O Redis password pode estar divergindo entre pods.

### Fix sugerido
```bash
# Verificar versão do ArgoCD
kubectl --kubeconfig ~/.kube/config-rbx -n argocd get deploy argocd-server \
  -o jsonpath='{.spec.template.spec.containers[0].image}'

# Restart completo de todos os componentes
kubectl --kubeconfig ~/.kube/config-rbx -n argocd rollout restart \
  statefulset/argocd-application-controller \
  deployment/argocd-server \
  deployment/argocd-repo-server \
  deployment/argocd-applicationset-controller \
  deployment/argocd-notifications-controller

# Aguardar e checar
sleep 60
kubectl --kubeconfig ~/.kube/config-rbx -n argocd get applications
```
